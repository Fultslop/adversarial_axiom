# Acceptance Test Plan: Closures (#20) — Phase C: Implicit Edge Case Discovery

**Library under test:** `@fultslop/axiom` (local build)
**Spec reference:** `docs/superpowers/specs/2026-04-13-closures-design.md`
**Test infrastructure:** `compileAndRun` helper — writes fixture to disk, compiles with `tspc`+transformer, optionally executes with `node`

Silent failure points identified by logic analysis.

---

| Test ID | Name | Category | Input Description | Expected Outcome | Risk Justification |
|---|---|---|---|---|---|
| **CL-C1** | Captured outer parameter used in `@pre` — no spurious warning | Implicit Edge Case | `export function makeAdder(base: number) { /** @pre base >= 0 */ return (x: number) => base + x; }` | No unknown-identifier warning for `base`. `@pre base >= 0` injected. | `base` is an outer parameter. Without `buildCapturedIdentifiers`, `base` is unknown to the inner known-identifier set (section 4.7). This is test case 6.6 from the spec. |
| **CL-C2** | Multiple captured outer parameters, all valid | Implicit Edge Case | Outer fn has `(a: number, b: number)`; inner closure uses both in `@pre a > 0` and `@pre b > 0` | Both tags injected; no warnings for `a` or `b`. | Verifies the full parameter iteration loop in `buildCapturedIdentifiers`. |
| **CL-C3** | Returned arrow: `@prev` with explicit `@prev` tag and captured outer param | Implicit Edge Case | Outer method returns closure; JSDoc: `@prev { count: state.count }` + `@post result >= prev.count`; `state` from outer param | `@post` injected with `prev` capture; `state` recognized as known identifier. | `@prev` on closures requires explicit tag (section 4.8). Tests both `@prev` and captured-identifier paths together. |
| **CL-C4** | Returned arrow: `@post` using `prev` without `@prev` tag — warning, dropped | Implicit Edge Case | Closure with `/** @post result >= prev.count */` and no `@prev` tag | Warning emitted: `'prev' used but no @prev capture available; @post dropped` with location `"outer > (anonymous)"`. No `@post` injected. | `resolvePrevCapture` returns `null` for non-method nodes. Silent contract injection with undefined `prev` would produce a `ReferenceError` at runtime. |
| **CL-C5** | Unknown identifier in inner `@pre` — warning emitted, tag dropped | Implicit Edge Case | Inner fn with `/** @pre ghost > 0 */` where `ghost` is neither a param nor a captured outer variable | Warning cites `ghost` as unknown and location `"outer > inner"`. `@pre` not injected; function body unchanged. | Prevents silent wrong-behavior: if unknown identifiers were not filtered, the compiled guard would reference an undeclared variable. |
| **CL-C6** | Inner fn with zero JSDoc tags — no mutation, no warning | Implicit Edge Case | Inner `FunctionDeclaration` with no `/** ... */` comment at all | Phase 2 skips the function; compiled output matches input except for outer-fn contracts. No `require` injected if outer also untagged. | Ensures Phase 2 does not touch unannotated nested functions, preserving developer intent and avoiding performance overhead. |
| **CL-C7** | `@invariant` on inner fn — ignored, no injection | Implicit Edge Case | Inner named fn with `/** @invariant this.value > 0 */` | No invariant check injected; no crash; optional #13 warning for `@invariant` on non-class node. | `@invariant` has no meaning on a closure (section 4.8 / 7). An implementation bug could attempt to call class-level invariant injection on a `FunctionDeclaration`, causing a transformer crash. |
| **CL-C8** | Expression-body arrow inside outer fn — normalised before rewrite | Implicit Edge Case | Outer fn body: `/** @pre x > 0 */ const fn = (x: number): number => x * 2;` (expression body, not block) | `@pre` injected correctly; compiled output contains block body with explicit `return`. No assertion loss from expression-body form. | `normaliseArrowBody` must fire for Rule B inner arrows just as it does for top-level arrows (section 4.5). If missed, `@post result` capture would be silently absent. |
| **CL-C9** | Inner arrow with `@post result` — requires explicit return type | Implicit Edge Case | `const fn = (x: number) => x * 2;` (no `: number` annotation) with `@post result > 0` | Warning emitted; `@post` dropped. | Same `filterPostTagsWithResult` rule applies to nested arrows. Missing return type at runtime would make `result` undefined. |
| **CL-C10** | Outer fn with contracts only (no inner tags) — Phase 2 is no-op | Implicit Edge Case | Outer fn has `/** @pre x > 0 */`; inner fn has no tags | Only one `ContractViolationError("PRE"` in output (for the outer). Phase 2 reference-equality check passes unchanged body through. | Ensures Phase 2 does not create a new body node unnecessarily, which would prevent the outer Phase 1 rewrite from being present in the output. |
| **CL-C11** | Inner fn with tags; outer fn has no contracts — `require` still injected | Implicit Edge Case | Outer fn has no JSDoc; inner fn has `/** @pre x > 0 */` | `require("@fultslop/axiom")` appears in compiled output. | The `require` injection is triggered by any contract guard being added to the file. If Phase 2 injects a guard without signaling `transformed.value = true`, the import is omitted and the guard throws `ReferenceError` at runtime. |
| **CL-C12** | Named inner fn: location string does not crash for variable-name outer | Implicit Edge Case | Outer fn is a `const`-assigned exported arrow at top level; inner named fn inside its body | Location string uses the outer variable name; does not produce `undefined > innerName`. | `buildLocationName` for outer must handle all outer-node forms that `tryRewriteFunction` accepts. An unhandled outer node type would produce a garbled or crashing location string. |
| **CL-C13** | Inner fn name shadows outer parameter name | Implicit Edge Case | Outer param is `x: number`; inner fn is `function x(n: number)` (shadowing) | Inner `@pre` uses `n` (inner param), not outer `x`; `buildCapturedIdentifiers` adds outer `x` to known set but inner param/name shadowing causes no collision error. | Name shadowing is a realistic TypeScript pattern. The two-phase known-identifier merge must not corrupt the inner param list with outer names that collide. |
| **CL-C14** | `const` binding before inner fn uses destructuring — captured correctly | Implicit Edge Case | `const { max, min } = opts;` precedes inner fn; inner `@pre x <= max && x >= min` | `max` and `min` recognized as captured identifiers (extracted via `extractBindingNames` on the destructure pattern). Tags injected, no warnings. | `buildCapturedIdentifiers` calls `extractBindingNames` on binding patterns (section 4.7). If only simple `Identifier` names are walked, destructured consts are silently missed. |
| **CL-C15** | Grandchild fn — no spurious injection into middle fn | Implicit Edge Case | Outer → `middle` (tagged: `/** @pre x > 0 */`) → `inner` (also tagged) | `middle`'s `@pre` is injected by Phase 2 of outer's rewrite. `inner`'s `@pre` is NOT injected (Phase 2 of `middle` does not fire because `middle` was not dispatched by `visitNode`). | Tests that Phase 2 is bounded to statements of the outer body — it does not recursively trigger another Phase 2 pass for the rewritten `middle` body. |
| **CL-C16** | Returned arrow is not the last statement — Rule C still applies | Implicit Edge Case | Outer fn: early `return (x: number): number => x;` with `@pre x > 0`, followed by unreachable code | `@pre` injected on the returned arrow; no crash; unreachable statements left intact. | Rule C matches any `ReturnStatement` with an arrow/fn-expression, not just the final one. A "last statement only" assumption would silently skip early-return closures. |
| **CL-C17** | `@pre` expression references both inner param and captured outer param | Implicit Edge Case | `/** @pre x > 0 && x < limit */` where `x` is inner param and `limit` is outer param | Both identifiers recognized; `@pre` injected; no unknown-identifier warnings. | Integration test for the merged known-identifier sets (`buildKnownIdentifiers` + `capturedIdentifiers`). |
| **CL-C18** | File-level `keepContracts` directive applies to inner contracts | Implicit Edge Case | `// @axiom keepContracts post` on line 1; outer fn and inner fn both have `@pre` and `@post` | Outer `@pre` stripped; inner `@pre` stripped; both `@post` retained. | `keepContracts` filtering applies globally to all contract injections in the file. Phase 2 must respect the same filter flag as Phase 1. |
